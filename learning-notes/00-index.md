# Thinking in Systems — Feynman Learning Notes

Plain-language reference notes for *Thinking in Systems: A Primer* by Donella
Meadows. Written in the Feynman spirit: **if you can't explain it simply with a
picture a 12-year-old would get, you don't understand it yet.**

Each chapter file follows the same shape:
1. **The one big idea** (a single sentence)
2. **A simple analogy** you can actually picture
3. **Core concepts**, jargon stripped out
4. **Where your understanding will crack** — the traps beginners fall into
5. **Stress-test questions** — explain it back in your own words

Read them in order the first time; use them as reference after.

## The spine of the whole book (memorize this)

> **Structure → Behavior → Events.**
> Most people react to *events* (today's headline). Smarter people spot
> *behavior* (the trend over time). Systems thinkers find the *structure* (the
> stocks, loops, and delays) that produces the behavior — because **structure is
> the only place real change happens.**

## The three words everything is built from

- **Stock** — an amount you could count if you froze time (users, water, trust).
- **Flow** — a rate that fills or drains a stock (per day/week/year).
- **Feedback loop** — when a stock's level reaches back and changes its own flows.

Everything else in the book is these three, combined and stacked.
